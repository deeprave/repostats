# Agent Guidance

Start with [openspec/project.md](openspec/project.md) for the current description of the `repostats` project and the intended architecture.

That file is the canonical summary of:

- what the project is trying to become
- what the interrupted refactor appears to have been attempting
- how to approach recovery work while preserving useful intent

OpenSpec artifact work is asynchronous with the development workflow:

- creating or updating proposals, designs, specs, and tasks does not require changing `.guide.yaml`
- do not switch issue or phase just to create or revise OpenSpec artifacts
- exception: when reflecting actual implementation progress for the current issue, keep that issue's artifacts in sync with what was done
